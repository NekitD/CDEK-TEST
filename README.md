# CDEK TEST:


### Требования
- Node.js (версия 14 или выше)
- npm (устанавливается вместе с Node.js)

### Инструкция по запуску

1. **Клонируйте репозиторий**
   ```bash
   git clone https://github.com/NekitD/CDEK-TEST
   cd CDEK-TEST/news

2. Установите зависимости, если ещё не установлены:
    ```bash
    npm install

3. Запустите:
    ```bash
    npm start


### Принятые решения
1. Архитектура компонентов

Проект разделен на 3 логических компонента:

    News.jsx - контейнерный компонент, отвечает за:
        - загрузку данных с API

        - Управление состоянием пагинации

        - Хранение метаданных (номер страницы, общее количество страниц)

        - Отображение заголовка и навигации

    newselem.jsx - презентационный компонент, отвечает за:

        - Отображение конкретной новости

        - Форматирование даты

        - Рендер рубрик с цветовой индикацией

        - Отображение лайков и просмотров

    skeleton.jsx - компонент-заглушка (взят из открытых источников: https://statuser.cloud/blog/skeleton-loading-v-react-bez-tailwind)

2. Управление состоянием

    Использован хук useState для хранения:

    - newsData - массив новостей

    - metaData - метаинформация (totalPages)

    - pageNum - текущая страница

    - loading/mloading - состояния загрузки

    Использован хук useEffect с зависимостью от pageNum для автоматической подгрузки данных при смене страницы

3. Пагинация

    - Реализована клиентская пагинация с помощью кнопок "←" и "→"

    - При смене страницы происходит очистка старых данных и показ скелетона

    - Кнопки навигации отображаются условно (первая/последняя страница)

4. Работа с API

    - Базовый URL API вынесен в константу API_BASE_URL

    - Функция GetNews выполняет асинхронный запрос и обновляет состояние

    - При смене страницы старые данные очищаются для предотвращения "мерцания"

5. Адаптивный дизайн

    - Использована методология БЭМ для именования классов

    - Реализованы медиа-запросы для мобильных устройств (max-width: 480px)

    Адаптация включает:

    - Изменение расположения картинки (сбоку → сверху)

    - Изменение отступов и размеров шрифтов

6. Форматирование данных

    convertMonthYear() - преобразует текущую дату в формат "Месяц Год"

    convertDate() - преобразует ISO дату в формат "День месяц часы:минуты"

    Цвет рубрик зависит от slug (top, cdek-active, school, skidki)

7. Стилизация

    Использована БЭМ-нотация для изоляции стилей

    Иконки (лайк, просмотр, стрелки) реализованы через текстовые символы.

8. Обработка загрузки

    При первичной загрузке отображается скелетон

    При смене страницы сразу показывается скелетон, старые данные очищаются

    Состояние loading предотвращает повторные рендеры
